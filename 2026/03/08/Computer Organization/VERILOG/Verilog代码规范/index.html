统一整个工程的命名方式。

信号极性为低有效用 n  后缀表示

多路选择器标明规格

魔数的命名

编写状态机的时候，各个状态一定要命名，之后调用的时候使用命名，而不是数字，减少代码中 magic number 的出现。使用parameter、localparam或者define。

信号仅在一个 always 块中赋值

组合逻辑采用 always @() 块或者 assign 编写

组合逻辑 always 块中仅使用阻塞赋值

组合逻辑 always 块中保证每个分支都进行赋值

如果组合逻辑中存在分支没有被赋值，那么实际综合中会综合出锁存器，FPGA 的底层基本组件并不存在锁存器。如果使用了条件语句 if 或者 switch，需要保证信号在每个可能的分支途径下都进行了赋值，其中 switch 语句一定要写 default 分支，并对信号赋值。

时序逻辑在 always @(posedge clock) 中实现

时序逻辑 always 块中仅使用非阻塞赋值

不要使用下降沿触发，特殊协议除外

不要使用非时钟 / 复位信号的边沿触发

时序逻辑中不要使用时钟信号

在时序逻辑中，请不要在敏感列表以外的地方使用时钟信号。

使用同步复位，而不是异步复位

对于 FPGA，请使用同步复位，因为异步复位容易受到毛刺的影响。

不要在内部模块中使用 inout

FPGA 内部的模块之间请不要使用 inout，仿真环境除外。

模块内部变量的定义和声明尽量统一

对于大模块的开发会使用大量的 wire 和 reg 变量，这些变量尽量统一在一个地方定义，后续增量开发在相应地方添加变量，而不是在需要使用的时候在旁边随手定义。另外，以免出现调用未声明变量的问题，可以添加宏定义 default_nettype。
// GOOD
wire RegWrite, MemtoReg, MemWrite, Busy, MdutoGRF;
wire [5:0] ALUop;
wire [4:0] Rs, Rt, Rd, WriteReg, shamt, ExcCode, ExcCodetmp, OvResult;
wire [31:0] Imm, RD1, RD2, SrcA, SrcB, ALUResult, WriteData, PC;
// Bad
wire RegWrite;
//...... 省略代码
wire [31:0] Imm, RD1, RD2, SrcA, SrcB;
//...... 省略代码

模块的实例化多换行

后期开发中，一个模块可能有几十个端口。如果模块实例化的时候，端口挤在一起，可读性就会非常差，建议一个端口换一行，input 和 output 分开写，例如：
// 模块实例化
// GOOD
test int_test(
.clk(clk),
.reset(reset),
.addr(addr),
.we(we),
.data_in(data_in),
.data_out(data_out)
);
// NOT GOOD
test int_test(.clk(clk),.reset(reset),.addr(addr),.we(we),.data_in(data_in),.data_out(data_out));

代码符号两侧空格部分规则

  • 单目运算符与变量间不添加空格，如 ~、!。
  • 双目运算符（除逗号外）和三目运算符两侧添加空格，如 +、=、<、&&。
  • 分号和逗号要紧附前面内容，不应添加空格。
  • 避免连续使用多个空格。

换行的使用

  • 对于不同逻辑的代码块建议换行分开，并加上相应注释区分
  • 对于同一逻辑，但是表达式复杂的语句，使用换行进行语义分割，如
    // GOOD
    assign d = (op == 0) ? a + b :
    (op == 1) ? a - b :
    (op == 2) ? a & b :
    a | b;

// BAD
assign d = (op == 0) ? a + b :(op == 1) ? a - b :(op == 2) ? a & b : a | b;

模块缩进

显式声明位宽

  • 使用常数时，声明数据位宽，避免连线时出现位宽不一致的 Bug。
  • 如果模块要使用数据的某一部分位，如 instruction[25:21]，使用 wire 变量直接截取，重新赋予一个合理的命名，减少字面量以增强可读性。
