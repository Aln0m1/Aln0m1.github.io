在 Verilog HDL 中，wire、reg 等数据类型默认都是无符号的。当你希望做符号数的操作时，你需要使用 **$signed()**。

假如我们有代码：
module comparator(
input clk,
input reset,
input [3:0] a,
input [3:0] b,
output res
);
assign res = a > b;
endmodule
于是我们编写了如下的测试程序，实现初始化 a = 4，b = 1，100ns 后 b 变为 -1。
经过测试，100ns之后res变成了0，而期望的结果是 res 始终为 1。
发生意外的原因在于：比较时 Verilog 将 a 和 b 默认视为无符号数，-1 会被认为是 15(4’b0000)。将比较代码修改为 assign res = $signed(a) > $signed(b); ，程序即可达到预期结果。![[Pasted image 20250922232052.png]]
当我们对无符号数和符号数同时操作时，Verilog 会自动地做数据类型匹配，将符号数向无符号数转化。例如执行 a > $signed(b) 时，a 是无符号数，$signed(b) 是符号数，Verilog 默认向无符号类型转化，得到的结果仍是无符号数的比较结果。

处理有符号数的原理

Verilog 对于符号的处理有些特殊，分为最外层表达式符号的确定与向内传播两个过程。也就是说先确定下来最终结果有无符号，再向内传播进行类型转换， 诸多诡异行为的罪魁祸首就是向内传播。

为了方便后续表述，让我们先定义两个概念。表达式和原子表达式，自决定表达式和上下文决定表达式。

  • 单独的运算数（比如常数，变量）是表达式。
  • 表达式与运算符的合法组合是表达式。
  • 定义了子表达式和原子表达式。
  • 自决定表达式的位宽和符号都是由自身决定的，和表达式所在的上下文无关。比如布尔表达式，该表达式的结果总是一位的无符号数。
  • 上下文决定表达式的位宽和符号既由自身又由上下文决定。比如 a + $signed(b) 的例子。它的符号不由自身决定，它取决于其子表达式。![[Pasted image 20250922233718.png]]

最外层表达式符号的确定

  • 自决定表达式的符号由自身决定。
  • 上下文决定表达式的符号和位宽由其子表达式决定。子表达式中有任一表达式是无符号则该表达式就是无符号的。
  • 子表达式有无符号怎么确定？当做最外层表达式来判定。实质递归。

内向传播

在用上述的规则确定好最外层表达式后，将最外层表达式的位宽和符号由外向内地传递给上下文决定的子表达式。这也是递归，直至遇到原子表达式时强制转换原子表达式的类型。

特殊情况

  • Verilog 语言规定关系表达式与等式表达式属于自确定与上下文决定的中间态，具体体现为结果是自确定的，但是它们的子表达式需要相互影响。
  • 移位运算符右侧的操作数总是被视为无符号数，对于符号的确定无影响，而是由运算符左侧的操作数和表达式的其余部分共同决定。
  • 对于三目运算符，其 ‘?’ 前的布尔表达式是自决定的表达式，不会对最外层表达式的符号造成影响。

处理有符号数的方法

一、保持运算数符号性一致

使用 $signed() 尽量使得运算数间有/无符号相同（移位运算是个例外），这样容易确定结果的符号。
