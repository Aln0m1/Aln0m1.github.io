Wire和Reg类型

reg 型变量通常用于时序逻辑，在 always 块中赋值，它会保持值直到下一次在 always 块中被更新，就像存储器一样，能记住之前的值。

  • Verilog规定：always块内声明的变量必须是reg类型。
    reg 类型在 Verilog 中是 “存储型变量” 的抽象，其核心特性是：在always块中被赋值，并且能保持其值直到被重新赋值，这与硬件中的 “触发器 / 寄存器” 功能完全对应：reg 只有在敏感列表触发时才更新，其他时候保持原值，就像存储器存储数据一样。
  • 值得注意的是，reg型变量不一定不会被综合成寄存器，也可能是ROM/RAM等存储器。
  • 虽然reg确实多用于时序逻辑，但也可以参与组合逻辑的构建。

把 wire 想象成导线，因为导线是用来传输信号的，没有存储功能，输入变了输出马上变，这和 wire 的连续赋值特性一致。

  • always块里不能给wire类型赋值。如果你想要这样做，应该把你需要的端口声明为reg类型或者干脆新建一个reg型端口（变量）

时序逻辑必须使用非阻塞赋值<=，而组合逻辑大多使用阻塞赋值=。
两者的差别就是：好比做饭，阻塞赋值必须要做完一道菜再开始做下一道菜，而非阻塞赋值可以让你在烧水的同时炒个鸡蛋。
从字面上理解：程序遇到阻塞赋值符号=的时候就被阻塞了，程序不执行完这一块就运行不下去；而非阻塞赋值<=不会把程序卡住，执行这一步的同时还会继续往下走，执行下面的指令。所以说非阻塞赋值通过这种方式实现了并行计算。

以下是一些细碎易错的知识点：

  • 使用未经声明的变量不会导致错误。变量未经声明即使用时默认为 1 位 wire 型变量。
  • 使用assign语句驱动时，左右两侧位宽不一致时，会对右侧变量进行扩展或截断，使之与左侧变量位宽相等。
  • 声明wire型向量：wire [num1:num2] a，那么num1，num2都是整数，num1和num2可以是负数，且其中较大值可以在前也可以在后。

数字字面量

Verilog 中的数字字面量可以按二进制（b 或 B）、八进制（o 或 O）、十六进制（h 或 H）、十进制（d 或 D）表示。数字的完整表达为 <位宽>’<进制><值>，如 10’d100。省略位宽时采用默认位宽（与机器有关，一般为 32 位），省略进制时默认为十进制，值部分可以用下划线分开提高可读性，如 16’b1010_1011_1111_1010。

Verilog 中除了普通的数字以外，还有两个特殊的值：x 和 z 。x 为不定值，当某一二进制位的值不能确定时出现，变量的默认初始值为 x。z 为高阻态，代表没有连接到有效输入上。对于位宽大于 1 的数据类型，**x 与 z 均可只在部分位上出现**。

字面量/常量/立即数的定义

integer

定义的数据类似int，32位，默认有符号数，常用于for循环

parameter

用于在编译时确认值的常量，通过形如 parameter 标识符 = 表达式; 的语句进行定义，如：parameter width = 8;。

在实例化模块时，可通过参数传递改变在被引用模块实例中已定义的参数。parameter 虽然看起来可变，但它属于常量，在编译时会有一个确定的值。
parameter产生的参数可以通过defparam强行修改。

parameter定义出的时模块级的常量，作用域仅限当前模块内部。它属于模块的一部分，会被综合工具识别为常量，不占用硬件资源。

localparam

用于定义模块内部的局部常量，其值在模块内部固定，无法被外部修改，是完全意义上的只读常量。

define

预编译指令。可以定义常量，可用undef取消定义。作用在整个程序范围。定义时不需要数据类型，使用define要在前面加上‘作为提示。本质上是一种文本替换，把你写的程序改了。

阻塞赋值与非阻塞赋值

module blocked_and_non_blocked(
input clk,
input a,
output reg b_blocked,
output reg c_blocked,
output reg b_non_blocked,
output reg c_non_blocked
);

// 非阻塞赋值
always @(posedge clk) begin
    b_non_blocked <= a;
    c_non_blocked <= b_non_blocked;
end
// 阻塞赋值
always @(posedge clk) begin
    b_blocked = a;
    c_blocked = b_blocked;
end

endmodule
