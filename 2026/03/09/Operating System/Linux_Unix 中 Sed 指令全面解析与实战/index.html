Linux/Unix 中 Sed 指令全面解析与实战

你希望全面、系统地掌握 Linux/Unix 中的 sed 指令，包括它的核心定义、语法规则、常用参数和实战场景，对吧？sed（流编辑器）是处理文本的核心工具，比 awk 更侧重行级别的替换、删除、插入、追加，我会从零基础讲起，用通俗的例子帮你吃透它的用法。

一、sed 指令核心定义

sed 是 Stream EDitor 的缩写，直译是「流编辑器」，它的核心特性：

  • 「流处理」：逐行读取文本（文件/管道输入），按预设规则处理，处理完立即输出，不默认修改原文件；

  • 「非交互式」：无需手动操作，通过命令/脚本批量处理文本；

  • 「核心能力」：行的查找、替换、删除、插入、追加，是日志清洗、配置文件修改、批量文本处理的高频工具；

  • 「兼容性」：所有 Linux/Unix 内置，无需额外安装。

二、sed 基础语法

sed 的语法分为「选项」「编辑指令」「输入文件」三部分，核心格式：

1. 核心组成拆解

部分 说明
可选参数 控制 sed 的执行模式（如 -n 静默输出、-i 修改原文件）
编辑指令 格式：[地址范围][编辑命令]，决定「处理哪些行」「做什么操作」
输入文件 要处理的文件；若省略，sed 读取标准输入（管道传递的内容）

2. 关键概念：地址范围（处理哪些行）

「地址范围」决定 sed 只处理哪些行，省略则处理所有行，常见写法：

地址范围写法 含义
5 仅处理第 5 行
1,5 处理第 1 到 5 行（闭区间）
3,$ 处理第 3 行到最后一行（$ 代表最后一行）
/正则/ 处理匹配「正则表达式」的行（如 /Hello/ 匹配含 Hello 的行）
1~2 处理奇数行（1~2 表示从 1 开始，步长 2）；2~2 处理偶数行
!/正则/ 处理不匹配正则的行（! 取反）

3. 核心编辑命令（做什么操作）

编辑命令是 sed 的核心，常用的只有十几个，新手先掌握以下 8 个：

命令 含义 常用场景
p 打印（Print） 提取指定行
d 删除（Delete） 删除指定行
s 替换（Substitute） 文本替换（最常用！）
a 追加（Append） 在指定行后插入内容
i 插入（Insert） 在指定行前插入内容
c 替换整行（Change） 把指定行整体替换为新内容
y 字符转换（Translate） 一对一字符替换（长度需一致）
= 打印行号 显示匹配行的行号

4. 常用可选参数

参数 含义 新手注意
-n 静默模式：只输出「被处理的行」，而非所有行（配合 p 命令使用） 不加 -n 用 p 会重复输出
-i 直接修改原文件（In-place） 慎用！建议先测试再用，可加 .bak 备份（如 -i.bak）
-e 执行多个编辑命令（多指令串联） 替代多次调用 sed
-f 从脚本文件读取编辑命令（适合复杂逻辑） 把指令写在 .sed 文件中
-r 启用扩展正则表达式（无需转义 `()、 、+` 等）
-E 等价于 -r（macOS 更常用） 跨平台兼容

四、sed 常见实战场景

  1. 批量修改配置文件：

把 nginx.conf 中所有 “8080” 替换为 “80”，并备份

sed -i.bak ‘s/8080/80/g’ nginx.conf

  1. 给代码加注释：

给 test.py 的 10-20 行开头加 # 注释

sed ‘10,20s/^/#/‘ test.py

五、新手避坑注意事项

  1. -i ** 参数慎用**：一定要先不加 -i 测试输出，确认无误后再用，建议加 .bak 备份；

  2. 替换分隔符：当替换内容含 /（如路径），改用 |、@ 等作为分隔符，避免冲突；

  3. 正则转义：默认 sed 用基础正则，()、|、+ 需转义（(、|、+），加 -r 可省略转义；

  4. 空行匹配：/^$/ 匹配空行，/^[ \t]*$/ 匹配含空格/制表符的空行（更严谨）；

  5. 原文件编码：若文件是 UTF-8 带 BOM，sed 可能处理异常，需先去除 BOM。

总结

  1. sed 核心是「逐行处理文本」，侧重行级的替换、删除、插入，s 命令（替换）是最常用的功能；

  2. 关键参数：-n（静默打印）、-i（修改原文件）、-r（扩展正则）、-e（多命令）；

  3. 核心语法：sed [参数] '地址范围命令' 文件，地址范围控制处理哪些行，命令控制做什么操作；

  4. 实战技巧：先测试输出（不加 -i），再修改原文件（加 -i.bak 备份），替换路径用自定义分隔符。
